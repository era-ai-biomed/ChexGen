# ChexGen — MedFMC ChestDR few-shot reproduction kit

Pipeline used in the ChexGen paper for the MedFMC ChestDR 19-class multi-label
**few-shot** experiment (1-/5-/10-shot, baseline vs. ChexGen-pretrained DenseNet121).
Built on top of [MedFM](https://github.com/openmedlab/MedFM) (mmcls 0.25.0).

## What's in here

```
medfmc_fewshot/
├── medfmc/                              # mmcls custom package (Chest19 dataset, AUC_multilabel hook)
├── tools/                               # mmcls train/test entry points
├── configs/
│   ├── _base_/                          # backbone, schedule, runtime, chest dataset config
│   └── densenet/medfmc_fewshot/         # 30 entrypoints: dense121_chest_256_<N>shot_exp<i>[_syn5].py
├── data_meta/                           # 30 N-shot train/val ann + test_WithLabel.txt (no images)
└── work_dirs/medfmc_fewshot/            # our 30 training runs (base + syn5, 3 shots × 5 exps each)
    └── <arm>/<N>_shot_exp<i>/           # *.log, *.log.json, frozen config snapshot
```

## Data preparation

### MedFMC ChestDR images

Obtain the official MedFMC ChestDR images (training set + held-out
`test_WithLabel.txt` test set) from the
[MedFMC 2023 challenge](https://medfm2023.grand-challenge.org/). Edit the root in
`configs/_base_/datasets/chest_256.py`:

```python
CHEST_ROOT = 'data/medfmc/chest/images'   # directory whose direct children are *.png
```

The filenames in `data_meta/chest_<N>-shot_*_exp<i>.txt` and
`data_meta/test_WithLabel.txt` should resolve under `<CHEST_ROOT>/<file>.png`.

### Labels (already in `data_meta/`)

- `chest_<N>-shot_train_exp<i>.txt` (15 files): N-shot train splits, N ∈ {1, 5, 10}, i ∈ {1..5}
- `chest_<N>-shot_val_exp<i>.txt`   (15 files): matching N-shot val splits
- `test_WithLabel.txt`              (1 161 lines): held-out ChestDR test set with labels

Format: `<filename>.png <19 comma-separated 0/1 labels>`, with the 19 columns in
ChestDR order (per `medfmc/datasets/medical_datasets.py:Chest19.CLASSES`).

### Backbone

```bash
mkdir -p weights
wget -O weights/densenet121_4xb256_in1k_20220426-07450f99.pth \
  https://download.openmmlab.com/mmclassification/v0/densenet/densenet121_4xb256_in1k_20220426-07450f99.pth
```

### Synthetic-pretrained init (for `*_syn5.py` only)

The `dense121_chest_256_<N>shot_exp<i>_syn5.py` configs load
`weights/sync5_pretrain.pth` — a DenseNet121 pretrained on 5× synthetic MIMIC-CXR
images (~225 k frames generated by ChexGen `finetuned_impression_512` over the
MIMIC p10–p18 captions). The checkpoint is available on request from the
authors. Place it at `weights/sync5_pretrain.pth` before running the syn5
configs.

## How to run

```bash
cd ChexGen/downstream/medfmc_fewshot
export PYTHONPATH=$PWD:$PYTHONPATH

# baseline (ImageNet init, no synthetic pretrain) — 15 runs
for SHOT in 1 5 10; do
  for EXP in 1 2 3 4 5; do
    python tools/train.py configs/densenet/medfmc_fewshot/dense121_chest_256_${SHOT}shot_exp${EXP}.py \
        --work-dir work_dirs/medfmc_fewshot/base/${SHOT}_shot_exp${EXP}
  done
done

# syn5 (ChexGen-pretrained init) — 15 runs
for SHOT in 1 5 10; do
  for EXP in 1 2 3 4 5; do
    python tools/train.py configs/densenet/medfmc_fewshot/dense121_chest_256_${SHOT}shot_exp${EXP}_syn5.py \
        --work-dir work_dirs/medfmc_fewshot/syn5/${SHOT}_shot_exp${EXP}
  done
done
```

## Reading our logs

Each `work_dirs/medfmc_fewshot/<arm>/<N>_shot_exp<i>/*.log.json` is one JSON
object per line. The `mode == "val"` rows give per-epoch overall `mAP` over the
19 classes; the reported per-run metric is the maximum over the 20 training
epochs:

```python
import json, glob
def best(p):
    rows = [json.loads(l) for l in open(p) if l.strip()]
    return max(r['mAP'] for r in rows
               if r.get('mode') == 'val' and 'mAP' in r)

for arm in ['base', 'syn5']:
    for shot in [1, 5, 10]:
        vs = [best(sorted(glob.glob(f'work_dirs/medfmc_fewshot/{arm}/{shot}_shot_exp{i}/*.log.json'))[-1])
              for i in range(1, 6)]
        print(arm, f'{shot}-shot', [f'{x:.2f}' for x in vs])
```

The released `evaluation` block uses `metric='AUC_multilabel'`, but our existing
logs only record `mAP` — AUC at the time was computed offline, separately, and
is not in these logs. Re-running with the released configs will produce AUC.
